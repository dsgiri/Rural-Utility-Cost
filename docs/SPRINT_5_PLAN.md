# Sprint 5 Strategy & Backlog

## Sprint Goal
Execute Phase 3 of the Practical Ranking Roadmap: **Cross-Cluster Sweeping and Footer Mapping**. This sprint focuses on establishing tight cross-linking between logically connected calculators in different domains and transforming the site footer into an authoritative navigation map.

## Sprint Backlog
- [x] **Task 1:** Create `/docs/SPRINT_5_PLAN.md` (This document)
- [x] **Task 2:** Overhaul the App Footer (`Layout.tsx`) taking it from a basic linear links layout to a semantic "Fat Footer" mapped into core clusters (Hubs, Top Tools, Company info).
- [x] **Task 3:** Cross-link calculators. For example, link "Energy Demand" (Utilities) into "Solar" and "Generator Runtime" directly where context fits.
- [x] **Task 4:** Cross-link property calculators. For example, add a call out in "Septic Tank" directing to "Well Drilling" (Water Planning Hub), as they are usually planned together.

## Rationale
Internal link equity (Pagerank) gets diluted if tools sit in isolated silos. The new Fat Footer ensures 100% crawler connectivity across standard hubs instantly. Furthermore, cross-cluster sweeping establishes semantic relationships between tools that share user intent (e.g., someone looking at septic is highly likely to look at wells).
